# Запуск Ceitnot — пошаговая инструкция для новичка

**Хотите один документ «от нуля до первого депозита на Sepolia»?**  
Используйте **[NOVICE-SEPOLIA.md](NOVICE-SEPOLIA.md)** — там всё по шагам с копируемыми командами.

---

## Что нужно заранее

1. **Node.js** (версия 18 или новее)  
   - Скачать: https://nodejs.org  
   - Проверить в терминале: `node -v` — должна показаться версия (например 20.x.x).

2. **Метамаск** (или другой кошелёк в браузере)  
   - Установить расширение: https://metamask.io  
   - Создать кошелёк и сохранить seed-фразу в надёжном месте.

3. **Терминал**  
   - Windows: PowerShell или «Терминал» в VS Code.  
   - Mac/Linux: «Терминал».

---

## Шаг 1. Открыть папку проекта

В терминале перейти в папку `aura` (где лежит этот проект):

```bash
cd F:\aura
```

*(На Mac/Linux путь будет свой, например `cd ~/aura`.)*

---

## Шаг 2. Запустить бэкенд (API)

2.1. Перейти в папку бэкенда:

```bash
cd backend
```

2.2. Установить зависимости (один раз):

```bash
npm install
```

2.3. Создать файл с настройками (один раз):

В папке `backend` уже есть файл-образец `.env.example`. Нужно сделать его копию с именем `.env` — в ней бэкенд будет искать настройки (порт, адрес контракта и т.д.).

**Что ввести в терминале** (вы должны быть в папке `backend`):

- **Windows (PowerShell):**
  ```powershell
  Copy-Item .env.example .env
  ```
- **Mac или Linux:**
  ```bash
  cp .env.example .env
  ```

После этого в `backend` появится файл `.env`. Менять в нём ничего не обязательно — для первого запуска подойдут значения по умолчанию.

2.4. Запустить сервер:

```bash
npm run dev
```

Должно появиться сообщение вроде: **Ceitnot backend running at http://localhost:3001**

- Окно с бэкендом **не закрывать** — пусть работает.
- Проверка: в браузере открыть http://localhost:3001/api/health — должна вернуться строка с `"status":"ok"`.

---

## Шаг 3. Запустить фронтенд (сайт)

3.1. Открыть **новое** окно терминала (бэкенд оставить запущенным в первом).

3.2. Перейти в папку проекта и затем во фронтенд:

```bash
cd F:\aura
cd frontend
```

3.3. Установить зависимости (один раз):

```bash
npm install
```

3.4. Запустить сайт:

```bash
npm run dev
```

Должно появиться что-то вроде: **Local: http://localhost:5173/**

3.5. Открыть в браузере: **http://localhost:5173**

- Должна открыться страница Ceitnot с блоком «Yield-Backed Credit Engine» и кнопкой **Connect wallet**.

---

## Шаг 4. Подключить кошелёк

4.1. На странице нажать **Connect wallet**.

4.2. Выбрать Метамаск (или другой кошелёк из списка).

4.3. В Метамаске подтвердить подключение к сайту.

4.4. После подключения появятся:
- ваш адрес в шапке (сокращённый);
- блок «Your position» и вкладки **Deposit**, **Borrow**, **Repay**.

---

## Что будет работать без контрактов

- Подключение кошелька.
- Показ адреса и интерфейса (Deposit / Borrow / Repay).
- Запрос к API здоровья бэкенда.

**Не будет** (пока не задеплоен контракт Ceitnot и не прописан его адрес):

- Реальные цифры по коллатералу, долгу и health factor.
- Реальные кнопки «Deposit», «Borrow», «Repay» (будет сообщение про настройку адреса контракта).

Как задеплоить контракт по шагам — смотри **[DEPLOY.md](DEPLOY.md)** (в той же папке `docs`).
- Статистика «Total collateral» / «Total debt» с блокчейна.

---

## Если хотите тестировать с реальным контрактом

1. Задеплоить смарт-контракты Ceitnot (через Foundry: `forge build`, затем скрипт деплоя на тестовую сеть).
2. В папке **backend** в файле **.env** указать:
   ```env
   CEITNOT_ENGINE_ADDRESS=0x...   # адрес прокси движка Ceitnot (`CeitnotProxy` → `CeitnotEngine`)
   ```
3. Перезапустить бэкенд (`npm run dev` в папке `backend`).
4. Во фронтенде перезагрузить страницу и переподключить кошелёк в нужной сети (Arbitrum или Base).

## Остановка

- В каждом окне терминала, где запущены бэкенд или фронт: нажать **Ctrl+C**.
- Сначала остановить фронт, потом бэкенд — как удобнее.

## Частые проблемы

| Проблема | Что сделать |
|----------|-------------|
| `npm: command not found` | Установить Node.js с https://nodejs.org и перезапустить терминал. |
| Порт 3001 или 5173 занят | Закрыть другое приложение на этом порту или в `.env` задать другой `PORT` (например 3002). |
| «Failed to fetch» на сайте | Убедиться, что бэкенд запущен на http://localhost:3001 и что в браузере открыт http://localhost:5173 (Vite проксирует `/api` на бэкенд). |
| Кнопка Connect не реагирует | Проверить, что расширение Метамаск установлено и разблокировано. |
| «Nonce #N пропущен» в кошельке (Rabby и др.) на Localhost | **Rabby:** меню (⋮) → **Clear Pending** — сбросит очередь; затем снова отправьте транзакцию. Другие кошельки: отменить pending для Localhost 8545. Либо перезапустить Anvil, задеплоить заново и в кошельке Clear Pending. |
| «Моделирование транзакций не поддерживается» в Rabby на Localhost | На Localhost 8545 Rabby не умеет симулировать транзакции. Подключите сайт через **MetaMask** (добавьте сеть Localhost 8545, RPC: http://127.0.0.1:8545, Chain ID: 31337) и делайте Deposit/Borrow из MetaMask. |

Если что-то не получается — пришлите точный текст ошибки из терминала или браузера (F12 → вкладка Console).
