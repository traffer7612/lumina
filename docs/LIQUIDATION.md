# Механика ликвидации (Ceitnot)

## Когда позиция может быть ликвидирована

Позиция считается **нездоровой** и доступна для ликвидации, когда **Health Factor (HF) < 1** (в WAD: HF < 1e18).

### Health Factor (HF)

Кросс-коллатеральный показатель по всем рынкам пользователя:

```
HF = Σ (collateralValue_i × liquidationThresholdBps_i / 10000) / Σ currentDebt_i
```

- **collateralValue_i** — стоимость коллатерала на рынке \(i\) в единицах долга (через оракул и vault).
- **liquidationThresholdBps_i** — порог ликвидации в базисных пунктах (например 8500 = 85%).
- **currentDebt_i** — текущий долг пользователя на рынке \(i\) (с учётом начисленных процентов).

Если общий долг равен нулю, HF считается бесконечным (позиция не ликвидируется).  
Чем ниже HF ниже 1, тем более «под водой» позиция.

---

## Кто может ликвидировать

**Любой адрес** может вызвать `liquidate(user, marketId, repayAmount)` при условии:

- HF пользователя \< 1;
- соблюдены ограничения по сумме (close factor, см. ниже);
- при включённом Dutch auction сначала был вызван `initiateLiquidation(user, marketId)` и при необходимости прошла часть окна `auctionDuration` (см. ниже).

Ликвидатор **вносит** в протокол токен долга (например USDC) в размере `repayAmount` и **получает** коллатерал (vault shares) с учётом штрафа ликвидации (часть уходит в протокол как комиссия).

---

## Как закрываются «плохие» долги

1. **Обычная ликвидация**  
   Ликвидатор погашает часть долга (`repayAmount`), получает эквивалент в коллатерале + премию (liquidation penalty). Долг и коллатерал позиции уменьшаются.

2. **Close factor**  
   За один вызов можно погасить не более заданной доли долга (например 50%), если HF выше порога полной ликвидации. Ниже этого порога можно погасить весь долг.

3. **Dutch auction (опционально)**  
   Если в реестре рынка `dutchAuctionEnabled == true`, после старта аукциона **эффективный liquidation penalty** растёт **линейно** от **0** до **`liquidationPenaltyBps`** за **`auctionDuration`** секунд. Сначала любой вызывает `initiateLiquidation(user, marketId)` (только при HF \< 1, один активный аукцион на позицию), затем `liquidate`: чем раньше ликвидация после старта, тем **меньше** бонус ликвидатору (меньше коллатерала на тот же `repayAmount`); по истечении `auctionDuration` действует полный штраф. Если `auctionDuration == 0`, после `initiateLiquidation` сразу применяется полный `liquidationPenaltyBps`.

4. **Bad debt (безнадёжный долг)**  
   Если после ликвидации коллатерала не хватает (позиция «под водой»), оставшийся долг списывается:
   - часть покрывается из **резервов** пула (totalReserves), если хватает;
   - непокрытая часть остаётся **bad debt** (протокол фиксирует событие `BadDebtRealized`).
   - Долг пользователя в этом рынке обнуляется, коллатерал обнуляется; протокол больше не ожидает возврата этой суммы от пользователя.

Таким образом, **плохие долги закрываются** за счёт: (1) ликвидаторов, погашающих долг в обмен на коллатерал; (2) резервов пула; (3) непокрытой части как признанного bad debt.

---

## События и мониторинг

- **Liquidated(user, liquidator, marketId, repayAmount, collateralSeized)** — успешная ликвидация.
- **BadDebtRealized(user, marketId, badDebtAmount)** — списание безнадёжного долга.

Рекомендуется настроить алерты на `BadDebtRealized` и на большие ликвидации (см. monitoring/README.md).

## 3 числовых сценария (простыми словами)

Ниже упрощённые примеры "на салфетке". В реальном протоколе точные значения зависят от цены оракула, valuePerShare vault и округлений.

### 1) Здоровая позиция (ликвидация невозможна)

Пусть у пользователя:
- collateralValue = 10,000 USDC
- liquidationThreshold = 80% (8000 bps)
- currentDebt = 7,000 USDC

Тогда:

```
HF = (10,000 × 0.8) / 7,000 = 8,000 / 7,000 = 1.14
```

Итог:
- HF > 1, позиция здорова;
- `liquidate(...)` вызвать нельзя.

### 2) Частичная ликвидация (HF < 1, но не "полный ноль")

Пусть:
- collateralValue = 10,000 USDC
- liquidationThreshold = 80%
- debt = 8,500 USDC

До ликвидации:

```
HF = (10,000 × 0.8) / 8,500 = 8,000 / 8,500 = 0.94
```

Позиция ликвидируема.  
Допустим:
- closeFactor = 50% -> за один вызов максимум `8,500 × 50% = 4,250` USDC погашения;
- liquidator делает `repayAmount = 2,000` USDC;
- effectivePenalty = 5%.

Упрощённо ликвидатор получает коллатерал на:

```
2,000 × (1 + 0.05) = 2,100 USDC эквивалента
```

Итог после операции (упрощённо):
- долг уменьшается: `8,500 -> 6,500`;
- коллатерал уменьшается на эквивалент изъятого;
- позиция может стать снова здоровой (или потребуются следующие ликвидации).

### 3) Bad debt (коллатерала не хватило)

Пусть рынок резко упал, и после изъятия всего коллатерала осталось:
- remainingDebt = 1,200 USDC
- totalReserves рынка = 700 USDC

Тогда:

```
coveredByReserves = min(1,200, 700) = 700
badDebtRealized   = 1,200 - 700 = 500
```

Итог:
- 700 покрывается из резервов;
- 500 фиксируется как `BadDebtRealized`;
- долг пользователя в этом рынке обнуляется, коллатерал уже ноль.

### Мини-шпаргалка по Dutch auction (на цифрах)

Если:
- `liquidationPenaltyBps = 500` (макс 5%)
- `auctionDuration = 3600` секунд

то эффективный penalty растёт линейно:

```
t = 0s      -> 0%
t = 1800s   -> 2.5%
t = 3600s+  -> 5%
```

Чем раньше после `initiateLiquidation` ликвидируют, тем меньше бонус ликвидатору и меньше изъятие коллатерала при том же `repayAmount`.
